## 要解决什么

书画展常见痛点是「只能远看，进不去、玩不了」。墨境流芳（作品集短名**墨境**）做成浏览器里的国风数字展馆：登录后漫游展区、工坊创作、课堂学习、个人空间归档，运营侧还能维护展品与看访问数据——答辩现场要的是**可跑通的闭环**，不是一张概念图。

## 架构怎么落

```
Vue 3 + TypeScript
  ├─ scene.ts（Three.js 主馆：山水 / 书法）
  ├─ three-jl（⑤灵兽区独立 WebGL，与主馆隔离）
  ├─ 水墨工坊 / 课堂 / 我的空间 / 运营后台
  └─ WebSocket 收工坊任务进度
        ↕
Spring Boot 3.2 + MyBatis-Plus + MySQL（dev 可用 H2）
  ├─ 展品 / 展区 / 用户 / 收藏 / 访问埋点
  └─ 异步 AI 任务（有 Key 走兼容生图 API，无 Key 本地水墨）
```

Docker Compose 可一键拉起 MySQL / Redis / EMQX / 前后端；本地默认 Redis、MQTT **可关**，不绑死演示。

## 真正有辨识度的点

### 1. 三展区 Web3D，而不是六区空壳

活跃展区收口到 **①山水 / ②书法 / ⑤灵兽**（③④已下线）。主馆用自研长场景脚本做行走、沉浸与后处理；灵兽区用独立 `three-jl` 粒子叙事，避免和主馆抢同一个 WebGL 上下文。

### 2. 工坊：异步 + 兜底，答辩不断链

上传 → 前端 Canvas 本地水墨预览 → 后端异步生成 → WebSocket 推进度 → 卷轴 3D 预览。有云端 Key 走 OpenAI 兼容生图；没有 Key 时 Java 本地水墨处理照样出结果。另可程序化导出 GLB / STL（简化卷轴几何，不是文物扫描建模）。

### 3. 课堂与运营是真模块

四门课 + 分步课纲（进度可落本地）；后台展品 CRUD、缩略图上传、ECharts 看板接真实 `visit_log`。这是「观展—创作—学习—运营」闭环里常被省略、却最能证明全栈的部分。

## 明确没做、也不该吹的

| 别写 | 事实 |
|------|------|
| 讯飞星火已驱动智能赏析 | 赏析主路径是预写文案 + 打字动效；Spark 方法写了但未接到主流程 |
| 六展区完整上线 | 仅三区可演示 |
| AR / 小程序 / 文创商城支付 | 文档明确本期不做 |
| Redis / MQTT 生产级依赖 | 可选，默认可关 |
| 完整 Spring Security | 轻量 Token 会话，适合演示不适合公网裸奔 |

诚实写边界，比包装「全 AI 数字孪生」更经得起问。

## 收获

Web3D 项目的工程难点往往不在「会不会调 Three.js」，而在**性能分级、资源隔离、演示 graceful degradation**。墨境让我把「现场断网 / 没 Key 也能讲完」当成一等需求——这和物联网项目里的 mock 上报是同一类思维。
