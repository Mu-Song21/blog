## 定位

安隅是作品集名；工程是 **若依二次开发**。目标：在成熟脚手架上，把物业常见台账做成可演示后台，而不是从零造轮子。

## 实际做了什么

在多模块工程中挂社区业务（cost / complaint / home / appointments / visitor / notice 等），典型能力：

- 列表 + PageHelper 分页 + 条件查询  
- 增删改  
- Excel 导出（POI）  
- 菜单 / 按钮权限走若依 RBAC（Security + JWT）  

表结构覆盖访客邀请（邀请码、楼栋单元房间）、缴费账单（金额/状态）、投诉工单、公告已读等。

## 没做、也不该吹的

| 不宜声称 | 事实 |
|----------|------|
| 第三方支付收银台 | 缴费是账单台账 |
| 真实门禁二维码通行 | 有邀请码字段，未对接门禁 |
| 自研权限框架 | 权限是若依能力 |
| UI 已全面换品牌 | 默认标题常仍是「若依管理系统」 |

部分目录有迭代痕迹（测试包、双包名）——展示抓主流程模块即可。

## 为什么仍值得写

很多实习的真实工作方式就是：**读懂脚手架 → 按规范加模块 → 配菜单权限 → 联调前后端**。你会 Maven 多模块怎么挂依赖、代码生成器生成后要改什么、不能盲信。

和青衿对比：安隅强调扩展效率；青衿强调自己接 Security/JWT。两个都写，能证明你会用框架，也懂框架底下的东西。分页与 MP 习惯见 [MyBatis-Plus 实践](/blog/18)。
