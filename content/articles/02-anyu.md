## 定位

安隅是作品集名；工程是 **若依二次开发**。目标：在成熟脚手架上，把物业常见台账做成可演示的后台模块，而不是从零造轮子。

## 我实际做的

在 `ruoyi-admin` 下挂社区相关 Controller，并拆出 cost / complaint / home / appointments / visitor / notice 等业务模块，典型能力：

- 列表 + PageHelper 分页 + 条件查询  
- 增删改  
- Excel 导出（POI）  
- 菜单/按钮权限走若依 RBAC  

表结构覆盖访客邀请（含邀请码、楼栋单元房间）、缴费账单（金额/状态）、投诉工单、公告已读等。

## 没做、也不该吹的

- **第三方支付网关**：缴费是账单台账，不是微信/支付宝收银台  
- **真实门禁二维码通行**：有邀请码字段，不是门禁系统对接  
- UI 默认标题仍是「若依管理系统」——品牌替换不是重点  

部分前端目录还有 `test` / 双包（community 与 smartcommunity）痕迹，说明迭代快——展示时抓主流程模块即可。

## 为什么仍值得写进作品集

因为这是很多实习的真实工作方式：**读懂脚手架 → 按规范加模块 → 配菜单权限 → 联调前后端**。你会：

- Maven 多模块怎么挂依赖  
- Security + JWT 在若依里如何生效  
- 代码生成器生成后要改什么、不能盲信  

## 和青衿的分工

| 安隅 | 青衿 |
|------|------|
| 站在若依肩膀上扩业务 | 自己搭 Security/JWT 教务后台 |
| 强调扩展效率 | 强调权限与校验理解 |

两个都写，能证明你会用框架，也懂框架底下的东西。
